How does lifetime elision reduce boilerplate in practice?
Rust's lifetime elision rules automatically infer lifetimes in ~60% of function signatures. A function like fn first(s: &str) → &str doesn't need explicit annotations because the compiler applies three elision rules.
How does this compare to Swift's ARC?
Swift's ARC handles memory at runtime via reference counting with a small per-retain/release cost. Rust achieves the same safety at compile time with zero runtime overhead — the tradeoff is satisfying the borrow checker upfront.
